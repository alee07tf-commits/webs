## Plan de INTERLINKING (distribución de autoridad)

### Principios aplicados (BigSEO)
1. **First Link Theory:** el primer enlace de una página a otra pesa el anchor. Cuidado con enlaces navegacionales.
2. **Retroactive linking:** cada artículo nuevo del blog enlaza a las páginas de servicio relevantes (primer link relevante = anchor de KW).
3. **Flujo de autoridad:** Home → Hubs → Hijas. Retroalimentación: hijas enlazan a la hub con anchor KW genérico.
4. **Página cornerstone:** /blog/agentes-ia-guia-completa/ recibe enlaces de TODO el cluster blog.

## Configuraciones técnicas base (para la nueva web)

### Hreflang (España + LATAM)
```html
<link rel="alternate" hreflang="es-es" href="https://zappyapps.tech/..." />
<link rel="alternate" hreflang="es-mx" href="https://zappyapps.tech/mx/..." />
<link rel="alternate" hreflang="es-ar" href="https://zappyapps.tech/ar/..." />
<link rel="alternate" hreflang="x-default" href="https://zappyapps.tech/..." />
```

Fase inicial: single-territory ES. Escalar a LATAM cuando haya tracción en España.

### Schema.org imprescindibles
- Organization + LocalBusiness (si aplica) en home
- Service en cada página /servicios/*
- Article + FAQPage en cada post de /blog/*
- BreadcrumbList global
- Product / Offer en /precios/
- HowTo en artículos paso a paso

### Robots + Sitemap
```
Robots: allow all. Disallow: /wp-admin/ /cart/ /checkout/ /my-account/
Sitemap.xml: separados — pages-sitemap.xml, services-sitemap.xml, blog-sitemap.xml
```

### Canonical
Self-referencing en todas las páginas. Prevenir duplicados en filtros/UTMs.
